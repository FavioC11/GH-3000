Objetivos de aprendizaje

  • Usar ghost text, chat, chat inline y agentes para tareas de codificación con flexibilidad.
  • Aplicar los 6 comandos de barra diagonal para obtener mejores respuestas con menos texto.
  • Aprovechar el contexto selectivo: archivos abiertos, @workspace y @terminal.
  • Completar un flujo real: crear ruta API → escribir prueba → documentar con agente.
Características avanzadas de GitHub Copilot Más allá del autocompletado: chat interactivo, comandos de barra diagonal, agentes de contexto y técnicas implícitas para corregir, documentar, generar y probar código en proyectos existentes.

Conceptos básicos — cómo activa Copilot sus sugerencias

Ghost text · texto fantasma↻ clic
Tab para aceptar · Esc para omitirSugerencias automáticas mientras escribes. Aparecen como texto atenuado. No requieren prompt: Copilot usa los archivos abiertos como contexto por defecto.
3 formas de indicar↻ clic
Comentario · Chat · Chat inline1. Escribe un comentario en el código → Copilot sugiere. 2. Abre el panel de chat y pregunta. 3. Ctrl+I / Cmd+I para chat inline en el punto del cursor.
Contexto por defecto↻ clic
Archivos abiertos en el editorPor defecto, Copilot usa todos los archivos abiertos como contexto adicional. No necesitas abrir muchos archivos; el chat puede consultar el workspace completo.

4 modos de interacción · nemotecnia: "Ghosts Chat Inline Agentes"

G — Ghost text↻ clic
Autocompletado pasivoSin prompt explícito. Copilot sugiere mientras escribes basándose en el contexto de los archivos abiertos. Tab para aceptar.
C — Chat panel↻ clic
Panel lateral dedicadoÍcono de chat en la barra lateral izquierda de VS Code. Conversación sobre código actual u otras preguntas de software. Historial de contexto en el panel.
I — Chat inline · Ctrl+I↻ clic
Sin cambiar de contextoCtrl+I (Win) / Cmd+I (Mac). Interacción en el punto exacto del código. Sugerencias más cercanas al código, sin abrir otro panel.
A — Agentes @↻ clic
Contexto especializado@terminal: ayuda con la salida y errores del terminal. @workspace: contexto de todo el proyecto. Permiten preguntas con un contexto más específico y potente.

6 comandos de barra diagonal · nemotecnia: "Desarrolladores Exigen Generar Hoy Óptimas Tareas"

D — /doc↻ clic
Agregar comentarios al códigoGenera documentación y comentarios para el código especificado o seleccionado. Útil para mantener el código autoexplicativo.
E — /explain↻ clic
Explicar el código seleccionadoObtiene una explicación detallada del fragmento de código. Ideal para aprender lógica existente o entender código de otros.
G — /generate↻ clic
Generar código desde una preguntaGenera código como respuesta a una solicitud específica. Equivale a un prompt libre pero con la intención marcada explícitamente.
H — /help↻ clic
Ayuda sobre Copilot ChatMuestra cómo usar el chat de Copilot, los comandos disponibles y cómo formular preguntas de forma más efectiva.
O — /optimize↻ clic
Mejorar tiempo de ejecuciónAnaliza y optimiza el rendimiento del código seleccionado. Propone algoritmos o estructuras más eficientes.
T — /tests↻ clic
Crear pruebas unitariasGenera pruebas unitarias para el código seleccionado. Combínalo con chat inline: selecciona el código → Ctrl+I → /tests → refina con el chat.

Técnicas avanzadas · nemotecnia: "Implícito Selectivo Iterativo"

I — Mensajes implícitos↻ clic
/fix sin escribir todoSelecciona código con error → Ctrl+I → escribe solo /fix. Copilot detecta automáticamente el problema y propone la corrección sin necesidad de describir el bug.
S — Contexto selectivo↻ clic
@workspace · @terminal@workspace: genera artefactos del proyecto completo (ej. Dockerfile). @terminal: explica errores de la consola. Copilot adapta las sugerencias al contexto del agente.
I — Iterar el prompt↻ clic
Replantear o empezar a escribirSi el resultado no es el esperado: sé más específico en el siguiente mensaje. O empieza a escribir código para que Copilot lo autocomplete con mejor contexto.

Flujo práctico completo — API + Tests + Docs

1 · Nueva ruta API↻ clic
main.py → Ctrl+I → promptAbre main.py → Ctrl+I → "Create a new route that exposes the cities of a country". Copilot genera el endpoint. Prueba y refina el prompt hasta obtener el resultado deseado.
2 · Crear prueba↻ clic
Seleccionar código → /testsSelecciona la ruta nueva → chat: "/tests help me to create a new test for this route using Spain". Si falla, describe el problema al chat: "This test is not quite right, it is not including cities that don't exist".
3 · Documentar con agente↻ clic
README.md → modo agenteAbre README.md → Copilot Chat con modo agente: "I want to document how to run this project so that other developers can get started quickly by reading the README.md file".

GitHub Codespaces — entorno preconfigurado en la nube

GitHub Codespaces↻ clic
VS Code en la nubeEntorno de desarrollador hospedado que funciona con VS Code. Preinstala dependencias, extensiones y configuraciones. Copilot viene preconfigurado. 60 h gratis/mes (2 núcleos).
⚠ Eliminar instancias↻ clic
Confirmar cambios antes de eliminarTras usar Codespaces: git add → git commit → git push → ve a github.com/codespaces → elimina la instancia. Si no confirmas, perderás todo el trabajo.

Resumen del módulo

Copilot ofrece 4 modos de interacción (Ghost text · Chat · Inline · Agentes) que se combinan para adaptarse a cualquier tarea en proyectos existentes.

6 slash commands: /doc · /explain · /generate · /help · /optimize · /tests — cada uno con una intención específica que mejora la respuesta sin prompts largos.

Técnicas clave: Mensajes implícitos (/fix sin describir el error) · Contexto selectivo (@terminal, @workspace) · Iteración del prompt cuando el resultado no es exacto.

Flujo real completo: crear endpoint API → generar prueba con /tests → documentar README con agente. Cada paso se refina iterando con el chat.
