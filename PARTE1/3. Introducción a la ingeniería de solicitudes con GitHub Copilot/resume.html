Infografía del módulo Ingeniería de Solicitudes con GitHub Copilot

Objetivos de aprendizaje

  • Comprender los principios de ingeniería de instrucciones y cómo Copilot aprende de tus prompts.
  • Aplicar estrategias avanzadas: role prompting y gestión del historial de chat.
  • Conocer el flujo interno de cómo Copilot procesa solicitudes y genera código.
  • Entender el manejo de datos en sugerencias de código vs. chat (retención, filtrado).
  • Comprender qué son los LLM, el fine-tuning y la optimización LoRA.
  • Crear prompts eficaces que minimicen ciclos de revisión.
¿Qué es la ingeniería de solicitudes? El proceso de crear instrucciones claras para guiar a Copilot a generar código sintáctica, funcional y contextualmente correcto. La calidad del output depende directamente de la calidad del prompt.

Las 4 S · los principios base de todo buen prompt

S — Single · Única↻ clic
Una tarea, un promptCentra cada solicitud en una única tarea bien definida. La claridad es fundamental para obtener respuestas precisas y útiles de Copilot.
S — Specific · Específica↻ clic
Instrucciones explícitasLas instrucciones explícitas y detalladas producen sugerencias más aplicables y precisas. Ej: "función Python que filtra números pares de una lista".
S — Short · Corta↻ clic
Concisa sin perder detalleMantén las solicitudes al grano. El equilibrio entre especificidad y brevedad garantiza claridad sin sobrecargar a Copilot ni complicar la interacción.
S — Surround · Enmarcada↻ clic
Contexto enriquecidoUsa nombres de archivo descriptivos y mantén abiertos los archivos relacionados. Copilot también usa las pestañas abiertas adyacentes como contexto adicional.

Procedimientos recomendados · nemotecnia: "Copilot Comprende Ejemplos Iterativamente"

C — Claridad suficiente↻ clic
Single + Specific aplicadosApunta siempre a la claridad. Una solicitud debe ser específica y centrada en una sola cosa. Si es ambigua, Copilot generará sugerencias genéricas.
C — Contexto con detalles↻ clic
Surround aplicadoEnriquece con contexto. Agrega comentarios en la parte superior del código, usa pasos numerados para detallar sin hacer el prompt largo.
E — Ejemplos para aprender↻ clic
Patrones concretosLos ejemplos bien diseñados ayudan a Copilot a comprender patrones, generando sugerencias más precisas. Ideal para código reutilizable y plantillas de prueba.
I — Iterar, no reescribir↻ clic
Diálogo, no monólogoSi el resultado no es el esperado: borra el código sugerido, enriquece el comentario con más detalles y ejemplos, y vuelve a preguntar. Cada iteración mejora.

Enfoques de aprendizaje · nemotecnia: "Sin Un Par de ejemplos"

Sin — Zero-shot↻ clic
Sin ejemplos específicosCopilot genera código basándose únicamente en su entrenamiento fundamental. Ideal para patrones comunes. Ej: un comentario que describe una función de conversión de temperatura.
Un — One-shot↻ clic
Un ejemplo de referenciaSe proporciona un único ejemplo. El modelo genera respuestas que siguen los patrones y convenciones del ejemplo. Crea implementaciones coherentes en el codebase.
Par — Few-shot↻ clic
Varios ejemplos · equilibrio óptimoEquilibrio entre zero-shot y fine-tuning. Genera implementaciones sofisticadas que manejan múltiples escenarios y casos extremos, reduciendo pruebas manuales.

Gestión del historial de chat · conversaciones largas = más costo

Resumir el contexto↻ clic
Condensar antes de continuar"Basándonos en nuestra discusión anterior sobre autenticación, ahora agrega limitación de velocidad…" Usa referencias concisas en lugar de repetir implementaciones completas.
Resetear para nuevas tareas↻ clic
Contexto fresco = mejor rendimientoPara nuevas características, empieza con detalles esenciales en lugar de continuar toda la conversación anterior. El historial completo puede confundir al modelo.
⚠ Costo de tokens↻ clic
2–3 PRU vs 1 PRU por turnoHistorial largo → 2–3 PRU por turno. Resumir o resetear → ~1 PRU. La ventana de contexto del chat es de 4.000 tokens. Desglosar en consultas más pequeñas mejora las respuestas.

Role prompting · nemotecnia: "Siempre Respeta Protocolos"

S — Experto en seguridad↻ clic
"Actúa como experto en ciberseguridad"Genera código con: saneamiento de entrada · protección contra ataques comunes · validación OWASP · procedimientos recomendados de seguridad.
R — Experto en rendimiento↻ clic
"Actúa como experto en optimización"Genera: algoritmos optimizados · estructuras de datos eficientes · implementaciones conscientes de memoria · consideraciones de escalabilidad.
P — Especialista en pruebas↻ clic
"Actúa como especialista en testing"Genera: cobertura exhaustiva · gestión de casos extremos · implementaciones ficticias (mocks) · pruebas de condición de error.

Flujo de proceso — 7 pasos de solicitud a sugerencia

1 · Transmisión + contexto↻ clic
HTTPS + FIMEnvío seguro vía HTTPS. Recopila: código antes/después del cursor, nombre del archivo, pestañas abiertas, estructura del proyecto. Técnica FIM para contexto bidireccional.
2 · Filtro de proxy↻ clic
Servidor Azure de GitHubEl proxy filtra el tráfico y bloquea intentos de manipular el sistema para revelar detalles internos del modelo. Primera línea de defensa.
3 · Filtrado de toxicidad↻ clic
Contenido + datos personalesElimina: discursos de odio, lenguaje ofensivo, contenido inapropiado. Filtra activamente datos personales (nombres, direcciones, IDs) para proteger la privacidad.
4 · Generación con LLM↻ clic
Modelo de lenguaje grandeLa solicitud filtrada y analizada se pasa al LLM, que genera sugerencias basadas en la comprensión del contexto. Garantiza relevancia y alineación con el proyecto.
5 · Validación de respuesta↻ clic
XSS · SQL · código públicoToxicidad post-generación. Chequeo de calidad: XSS, SQL injection. Filtro opcional: si la respuesta coincide con código público en >~150 caracteres, se descarta.
6 · Entrega + feedback↻ clic
Bucle de retroalimentaciónSolo pasan los filtros → se entrega. Copilot aprende de: sugerencias aceptadas (amplía conocimiento) · rechazos y modificaciones (mejora el modelo).
7 · Repetición continua↻ clic
El modelo aprende contigoEl proceso se repite con cada prompt. Copilot acumula datos de contexto e interacciones para mejorar progresivamente su comprensión de tu estilo y necesidades.

Control de datos — qué conserva Copilot y qué descarta

Sugerencias de código↻ clic
No se conservan · se descartanCopilot no conserva ninguna solicitud de código para entrenar modelos fundamentales. Las descarta una vez devuelta la sugerencia.
Chat de Copilot · 28 días↻ clic
Retención temporalGitHub conserva solicitudes, sugerencias y contexto auxiliar del chat durante 28 días. Aplica también a CLI, móvil y chat en GitHub.com.
Opt-out individual↻ clic
Control del usuarioLos suscriptores individuales pueden optar por no compartir sus solicitudes con GitHub para el refinamiento del modelo fundamental. Es una opción de privacidad disponible.

LLM y optimización LoRA

¿Qué son los LLM?↻ clic
Modelos de lenguaje grandeRedes neuronales con millones/billones de parámetros entrenados en grandes volúmenes de texto. Comprenden contexto, son versátiles y aplicables en múltiples dominios.
LLM en Copilot↻ clic
Contexto = todos los archivos abiertosCopilot considera no solo el archivo actual sino también todos los archivos y pestañas abiertos en el IDE para generar finalizaciones precisas y relevantes.
Fine-tuning↻ clic
Adaptar LLM a tareas específicasEntrenar el modelo en un dataset pequeño específico (destino) aprovechando los parámetros de un modelo grande preentrenado (origen). Mejora el rendimiento en dominio.
LoRA · Low-Rank Adaptation↻ clic
Trabajar más listo, no más difícilAgrega capas entrenables pequeñas a cada capa del modelo sin cambiar el original. Ahorra tiempo y recursos vs. fine-tuning completo. Supera a adaptadores y ajuste de prefijos.

Resumen del módulo

La ingeniería de solicitudes es el arte de guiar a Copilot con precisión. La calidad del output depende directamente de la calidad del prompt: aplica las 4 S (Single · Specific · Short · Surround) en cada interacción.

Estrategias clave: Zero/One/Few-shot para calibrar ejemplos · Role prompting para código especializado · Gestión del historial para optimizar tokens.

Internamente: cada prompt pasa por 7 pasos (transmisión segura → proxy → toxicidad → LLM → validación → entrega → repetición). El chat conserva datos 28 días; las sugerencias de código se descartan.

LoRA permite adaptar los LLM a dominios específicos sin reentrenar todo el modelo, haciendo a Copilot más preciso para tu contexto de codificación.
