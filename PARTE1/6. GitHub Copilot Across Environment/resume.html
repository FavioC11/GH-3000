Infografía del módulo GitHub Copilot Across Environment

Objetivos de aprendizaje

  • Usar sugerencias automáticas, panel múltiple y adaptación de estilos en el IDE.
  • Proporcionar contexto a Copilot mediante comentarios (inline, block, docstrings, TODO, API).
  • Mejorar las respuestas del chat con referencias de ámbito (#file, @terminal), slash commands y agentes.
  • Aprovechar Copilot en GitHub.com para PRs, issues, revisión de código y GitHub Actions.
  • Interactuar con la CLI de Copilot en modo interactivo y one-shot.
Copilot en todos tus entornos Desde el IDE hasta la terminal, pasando por GitHub.com: Copilot se adapta a donde trabajas — autocompletado, chat conversacional, asistencia en PRs y comandos de shell, todo integrado.

Adaptación al estilo · nemotecnia: "Ingenieros Construyen Formatos Comprensibles con Patrones"

I — Implementación de métodos↻ clic
Sigue tu estilo de codificaciónAl empezar a escribir un nombre de método, Copilot sugiere la implementación completa siguiendo el estilo de codificación establecido en el proyecto.
C — Convenciones de nombres↻ clic
camelCase · snake_case · PascalCaseRecoge las convenciones de nomenclatura preferidas para variables, funciones y clases y las aplica consistentemente en las sugerencias.
F — Formato↻ clic
Sangría · llaves · espaciadoSe adapta al estilo de sangría (tabs vs espacios), colocación de llaves y otras preferencias de formato presentes en el archivo activo.
C — Estilo de comentario↻ clic
Inline · bloque · docstringsImita el estilo de comentario preferido: comentarios en línea, en bloque o cadenas de documentación (docstrings), según lo que uses en el código.
P — Patrones de diseño↻ clic
Alinea sugerencias con el proyectoCuando el proyecto usa consistentemente ciertos patrones (Factory, Repository, Observer…), Copilot sugiere código que se alinea con esos patrones.

Tipos de comentarios usados como contexto · nemotecnia: "Ingenieros Buenos Documentan Todo Adecuadamente"

I — Comentarios inline↻ clic
Junto a líneas específicasExplicaciones breves al lado del código. Copilot las usa para entender la intención inmediata y sugerir la siguiente línea o completar la lógica.
B — Bloque de comentarios↻ clic
Descripción de función o claseExplicaciones más largas previas a una función o clase. Copilot puede sugerir la implementación completa basándose en la descripción del bloque.
D — Docstrings↻ clic
Documentación formal (Python, JS)Cadenas de documentación formales. Copilot las lee para entender parámetros, tipo de retorno y propósito, generando implementaciones más precisas.
T — Comentarios TODO↻ clic
Notas de implementación futuraNotas sobre mejoras o implementaciones pendientes. Copilot las usa para sugerir código que complete la tarea marcada como pendiente.
A — Documentación de API↻ clic
Uso y parámetros de funcionesComentarios que describen uso, parámetros y valores de retorno de métodos. Copilot genera implementaciones que respetan el contrato documentado.

Chat en el IDE — referencias de ámbito, agentes y modelos

3 usos principales del chat↻ clic
Generar · Depurar · Explicar1. Generar código complejo (regex, SQL, algoritmos). 2. Depurar: analiza errores y sugiere correcciones. 3. Explicar: divide código complejo en términos sencillos.
#file · @terminal↻ clic
Enfocar el contexto exacto#file:controller.js → Copilot se centra en ese archivo. @terminal → analiza la salida de la consola. Mejoran la precisión sin escribir prompts largos.
Agentes IDE↻ clic
@terminal · @vscode · /new@terminal: comandos y errores de consola. @vscode: configuración del IDE. /new: genera proyecto completo desde cero según requisitos.
Modelos · 1 PRU vs 2 PRU↻ clic
Estándar vs PremiumEstándar (GPT-4o): 1 PRU · tareas rutinarias, depuración básica. Premium (o1-preview, o1-mini): 2 PRU · análisis complejo, arquitectura, multihilo. Agentes complejos: 2–5 PRU.

Copilot en GitHub.com · nemotecnia: "Exploramos PRs Investigando Resultados con Acciones"

E — Exploración del repo↻ clic
Explicar · Resumir · DocumentarExplica secciones de código complejas, genera resúmenes del propósito del repositorio y crea/mejora README, docs de API y comentarios de código.
P — Asistencia en PRs↻ clic
Resúmenes · Revisión · ConflictosGenera resúmenes completos de cambios (1–2 PRU). Sugiere mejoras antes de la revisión formal. Ayuda a resolver conflictos de merge. Actualiza documentación automáticamente.
I — Gestión de issues↻ clic
Analizar · Idear · ReproducirDivide issues complejos en tareas accionables, genera posibles soluciones con enfoques de implementación, ayuda a crear pasos claros de reproducción de errores.
R — Revisión de código↻ clic
Seguridad · Rendimiento · CalidadGenera comentarios de revisión con sugerencias específicas. Identifica vulnerabilidades de seguridad y malas prácticas. Sugiere optimizaciones de rendimiento (1–3 PRU).
A — GitHub Actions · errores↻ clic
Analizar logs · Sugerir soluciónExamina los logs de flujos con errores, identifica la causa raíz, proporciona recomendaciones específicas y mejores prácticas para mejorar la confiabilidad del workflow.

CLI de Copilot · terminal directo

Instalación↻ clic

2 modos de uso↻ clic

Slash commands clave↻ clic
Solo controlan la sesión/explain · /suggest · /revise · /feedback · /exit · /model · /theme · /skills · /mcp · /list-dirs. No reemplazables por lenguaje natural.
Flujo típico↻ clic
Explicar → Sugerir → Revisar1. Explica un comando desconocido. 2. Sugiere comando para una tarea en lenguaje natural. 3. Refina con prompt de seguimiento. 4. Revisa siempre antes de ejecutar.

Resumen del módulo

Copilot opera en 3 entornos: IDE (autocompletado + chat + agentes) · GitHub.com (PRs, issues, revisión, Actions) · CLI (explicar y sugerir comandos de shell).

En el IDE: se adapta a 5 aspectos de tu estilo (implementación, convenciones, formato, comentarios, patrones) y usa 5 tipos de comentarios como contexto. Slash commands + #file/@terminal mejoran la precisión.

Modelos: Estándar (1 PRU) para tareas rutinarias · Premium (2 PRU) para análisis complejo. Agentes avanzados pueden consumir 2–5 PRU.

CLI: modo interactivo (copilot) o one-shot (copilot -i "..."). Los slash commands controlan la sesión y no son reemplazables por lenguaje natural.
