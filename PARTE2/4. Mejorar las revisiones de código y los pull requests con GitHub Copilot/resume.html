Objetivos del módulo
  • Identificar las características clave que Copilot agrega al proceso de revisión de código.
  • Solicitar e interpretar revisiones de Copilot en GitHub.com y en el IDE.
  • Personalizar el comportamiento con instrucciones específicas de repositorio y ruta.
  • Automatizar revisiones con conjuntos de reglas a nivel de cuenta, repo y organización.
  • Entender las PRUs y optimizar su uso para máximo valor.

Revisiones de código con GitHub Copilot

Copilot actúa como revisor colaborativo — detecta problemas, sugiere mejoras y redacta comentarios, sin reemplazar el juicio humano.

5 características clave — mnemónico: "RSERI: Resúmenes, Seguridad, Explicaciones, Redacción, IDE"
R — Resúmenes de PR

Descripciones automáticas de cambios

↩ detalle

Redacta automáticamente descripciones de PR que incluyen:

✅ Resumen claro de los cambios.

✅ Lista de archivos afectados.

Los revisores comienzan con contexto, no con estimaciones.

Usa el ícono de Copilot en el editor de descripción del PR.

S — Seguridad

Detección y corrección de vulnerabilidades

↩ ejemplo

Integrado con el análisis de código de GitHub; detecta vulnerabilidades en todos los lenguajes.

Ejemplo JS: detecta eval() con entrada del usuario → comenta: "Puede causar inyección de código. Reemplaza por JSON.parse()".

Ofrece parche inline alineado con las guías de seguridad del repositorio.

E — Explicaciones + R — Redacción

Línea a línea y comentarios de revisión

↩ cómo ayudan

Explicaciones: resalta código y pide a Copilot que explique la funcionalidad. Ideal para entender código desconocido rápidamente.

Redacción: genera comentarios de revisión basados en procedimientos recomendados o directrices del equipo. Comentarios claros y accionables.

Formato de datos en PRs

Tablas y métricas con buen formato

↩ ejemplo

Detecta tablas con mal formato en la descripción del PR.

Aplica las directrices de Markdown de .github/copilot-instructions.md.

Propone versión corregida lista para pegar con un solo clic.

Copilot actúa como revisor automático de estilo, no como agente de codificación.

I — IDE local

Detecta problemas antes del PR

↩ ventajas

Revisa cambios en VS Code o JetBrains antes de hacer commit.

Detecta infracciones de estilo, brechas de seguridad y problemas de procedimientos recomendados antes del ciclo de revisión formal.

Reduce el ruido en la revisión posterior.

Con PRUs: análisis más profundo, diferencias más grandes, instrucciones personalizadas.

Multi-idioma

Revisión fuera de tu área de experiencia

↩ cómo ayuda

Copilot resalta automáticamente áreas que no siguen procedimientos recomendados en el lenguaje específico.

Genera mejoras alineadas con las convenciones del lenguaje.

Proporciona comentarios más sólidos incluso fuera de tu área principal.

Ejemplo: sugiere refactorización limpia en Ruby sin que seas experto en Ruby.

Revisión en GitHub.com — flujo de 4 pasos + límites
Flujo de revisión

4 pasos en GitHub.com

↩ pasos

1. Abre o crea un PR en el repositorio.

2. En menú Revisores → selecciona Copilot.

3. Espera ~30 segundos a que Copilot analice.

4. Lee los comentarios en las líneas pertinentes y aplica sugerencias.

Los comentarios se ven y comportan como los de compañeros de equipo.

Aplicar sugerencias

Correcciones desde la interfaz del PR

↩ cómo

Confirma correcciones directamente desde la interfaz del PR.

Para comentarios entre pares: pide a Copilot que genere la solución.

Ejemplo: "Sugerir corrección para: Reemplaza exec() por una función más segura" → Copilot propone subprocess.run().

Desarrollador prueba localmente y confirma que las pruebas pasan antes de hacer merge.

Límites del rol

Asesoramiento, no decisión final

↩ qué NO hace

❌ No aprueba ni rechaza PRs.

❌ Sus comentarios no cuentan para aprobaciones requeridas.

❌ No reemplaza decisiones arquitectónicas complejas.

✅ Detecta problemas temprano.

✅ Genera sugerencias accionables.

✅ Acelera verificaciones rutinarias.

Confía en revisores humanos para las decisiones finales.

Personalización e instrucciones — archivos de guía para el revisor
copilot-instructions.md

Reglas globales del repositorio

↩ ejemplos de reglas

Ruta: .github/copilot-instructions.md

Ejemplo de reglas:

"Céntrate en la seguridad y evita la interpolación de cadenas no segura."

"Asegura que las funciones tienen docstrings con parámetros y tipos de retorno."

Copilot aplica estas reglas automáticamente a todas las revisiones del repo.

Instrucciones por ruta

.github/instructions/*.instructions.md

↩ configuración applyTo

Crear en .github/instructions/ archivos *.instructions.md.

Frontmatter con applyTo + glob:

Modelos Ruby: applyTo: "app/models/**/*.rb"

TypeScript: applyTo: "**/*.ts,**/*.tsx"

Todo el repo: applyTo: "**"

Copilot aplica esas instrucciones solo a los archivos coincidentes.

3 niveles de automatización — mnemónico: "CRO: Cuenta, Repositorio, Organización"
C — Cuenta personal

Pro / Pro+ · todos los PRs propios

↩ pasos

1. Foto de perfil → Su Copilot.

2. Busca "revisión de código automática de Copilot".

3. Lista desplegable → Habilitado.

Todos los PRs que abras incluirán automáticamente revisión de Copilot.

Solo disponible en Copilot Pro y Copilot Pro+.

R — Repositorio

Conjunto de reglas de rama

↩ pasos

1. Repo → Configuración → Código y automatización → Reglas → Conjuntos de reglas.

2. Nuevo conjunto de reglas de rama.

3. Nombre, Estado: Activo, ramas de destino.

4. Marcar "Requerir revisión de PR antes de combinar".

5. Activar "Solicitar revisión de PR de Copilot".

Opcional: habilitar "Requerir resolución de conversación antes de combinar".

O — Organización

Multi-repositorio a escala

↩ pasos

1. Foto de perfil → Sus organizaciones → Configuración.

2. Barra lateral → Repositorio → Conjuntos de reglas.

3. Nuevo conjunto de reglas de rama.

4. Agregar repositorios destino con patrones glob (p. ej. *servicio).

5. Definir ramas, habilitar revisión de Copilot → Crear.

Garantiza estándares coherentes y reduce tiempos de revisión en toda la org.

PRUs — mnemónico: "PERE: Planea, Estratégico, Refina, Escala"
¿Qué son las PRUs?

El "engranaje adicional" de Copilot

↩ cuándo se consumen

Tokens que desbloquean el análisis profundo de Copilot.

No consumen PRU: refactorizaciones ligeras de una línea.

Consumen PRU: revisar PR de 1500+ líneas en varios archivos, aplicar copilot-instructions.md, análisis de seguridad y estilo completo.

Cada vez que Copilot comenta en un PR como revisor: 1 PRU.

Impacto medible

Métricas antes/después

↩ qué medir

Plazo de PR: tiempo de apertura a combinación. (Ejemplo: 3 días → 1 día)

Calidad: reducción de problemas post-merge detectados por otras herramientas.

Experiencia del dev: ¿las revisiones son más rápidas y claras?

Ejemplo real: -30% en uso de PRUs al reservarlas solo para código de producción.

Optimización PERE

Presupuestar y maximizar valor

↩ estrategias

P — Planear: configura alertas al 75%, 90% y 100% del uso mensual.

E — Estratégico: reserva PRUs para cambios grandes o de alto riesgo en producción.

R — Refinar: mensajes claros y específicos reducen reintentos y PRUs desperdiciadas.

E — Escalar: si el equipo supera constantemente el límite, considera un plan Copilot superior.

Resumen

Características RSERI: Resúmenes de PR automáticos, Seguridad (detección + parche inline), Explicaciones línea a línea, Redacción de comentarios, IDE local antes del PR.
GitHub.com: Agregar Copilot como revisor → 30s → comentarios en líneas. Asesoramiento, no aprueba ni rechaza. Aplica sugerencias + pruebas locales antes de merge.
Personalización: copilot-instructions.md (global) + .github/instructions/*.instructions.md con applyTo por ruta glob.
Automatización CRO: Cuenta (Pro/Pro+, todos los PRs), Repositorio (conjunto de reglas de rama), Organización (multi-repo con patrones glob).
PRUs PERE: Planea alertas → Estratégico (alto riesgo) → Refina mensajes → Escala si necesario.
