Copilot Cloud Agent

Un compañero de equipo autónomo que trabaja en segundo plano dentro de GitHub: crea ramas, redacta código y abre PRs mientras tú revisas.

¿Qué es y qué puede hacer? — mnemónico: "BFPDD: Bugs, Features, Pruebas, Docs, Deuda"
Disponibilidad

Planes y repositorios compatibles

↩ detalle

Planes: Pro, Pro+, Business, Enterprise.

Repos: todos los hospedados en GitHub, excepto cuentas de usuario administradas o donde esté deshabilitado.

Idioma principal: inglés.

Tareas BFPDD

Qué puede hacer el agente

↩ lista

B — Bugs: corregir errores y regresiones.

F — Features: implementar nuevas características incrementales.

P — Pruebas: mejorar cobertura o generar pruebas faltantes.

D — Docs: actualizar o crear documentación.

D — Deuda: abordar deuda técnica y pendientes.

2 formas de delegar

Cómo asignar trabajo al agente

↩ métodos

1. Asignar un issue: en GitHub.com, Mobile, API GraphQL o CLI (gh issue edit).

2. Pedir desde chat: panel Agentes en GitHub, Copilot Chat, IDE u otra herramienta con MCP/Raycast (macOS).

Cuando termine, solicita tu revisión y responde a @copilot en comentarios del PR.

Cloud Agent vs. Modo Agente (IDE) vs. Asistentes tradicionales
Asistente IDE tradicional

Copilot inline / Chat local

↩ características

Ayuda a escribir código localmente.

Pasos manuales tuyos: crear ramas, commit, push, descripción PR.

Sesión privada, no visible para el equipo.

Sin trazabilidad en el historial de GitHub.

Modo Agente (IDE)

Copilot Edits en VS Code / VS

↩ características

Modificaciones locales autónomas directamente en la sesión del IDE.

Edita múltiples archivos dentro del workspace local.

Requiere que tú hagas push y abras el PR.

No se ejecuta en la nube de forma asíncrona.

Cloud Agent

Agente asíncrono en GitHub

↩ características

Todo ocurre como commits en GitHub (trazable).

Automatiza: rama copilot/, mensajes de commit, borrador de código.

Visible en registros de sesión e historial de PR para todo el equipo.

Iteras orientando con comentarios en el PR, no en sesión local.

Modelo de seguridad — mnemónico: "GERPC: Gobernanza, Entorno, Rama, Permisos, Coautoría"
G — Gobernanza + E — Entorno

Control org y sandbox seguro

↩ detalle

Gobernanza: la config de org/empresa rige la disponibilidad; todas las directivas de seguridad siguen aplicándose.

Entorno: sandbox aislado en Actions con acceso a Internet con firewall y acceso de solo lectura al repositorio.

R — Ramas + P — Permisos

Solo copilot/* y solo escritura

↩ detalle

Ramas: solo puede crear e insertar en ramas copilot/; las protecciones de rama y comprobaciones necesarias siguen activas.

Permisos: solo responde a usuarios con acceso de escritura; se ignoran comentarios de otros usuarios.

C — Coautoría + Aprobaciones

Atribución clara e independencia

↩ detalle

Coautoría: todos los commits son co-autoría con el desarrollador que asignó la tarea. Atribución clara.

Aprobaciones: los PRs del agente son siempre borrador. El solicitante NO puede aprobarlos; se conservan las reglas de "revisiones requeridas".

3 Riesgos + Limitaciones conocidas
3 Riesgos — "III"

Inserción · Información · Inyección

↩ mitigaciones

Inserción de código: solo ramas copilot/, aprobación obligatoria, solicitante no puede autoaprobar.

Información confidencial: firewall por defecto restringe acceso a Internet.

Inyección de prompts: caracteres ocultos (HTML comments, etc.) se filtran antes de pasar la entrada al agente.

Limitaciones conocidas

Flujo de trabajo y compatibilidad

↩ lista

Solo puede cambiar el mismo repositorio del issue/PR.

Abre exactamente 1 PR por tarea.

No firma commits; no usa ejecutores autohospedados (solo Ubuntu x64).

No respeta exclusiones de contenido ni la directiva "Bloquear código público".

El modelo de IA lo selecciona GitHub; no se puede cambiar.

Solo repos hospedados en GitHub.

Ciclo de trabajo del agente — mnemónico: "ARIAC: Asigno, Rastro, Itero, Apruebo, Combino"
A — Asignar

Cómo delegar el trabajo

↩ métodos

GitHub.com: issue → Asignar → Copilot.

Lista de issues, Projects o Mobile: misma asignación.

CLI: gh issue edit --add-assignee @copilot

API GraphQL: mutación replaceActorsForAssignable con el Bot ID de Copilot.

El agente recibe el título, descripción y comentarios existentes al momento de asignación.

R — Rastrear + I — Iterar

Señales de progreso y feedback

↩ señales

👀 reacción al issue = confirmación de recepción.

Rama copilot/ + borrador de PR abierto automáticamente.

Sesión activa: mensajes de estado en el cuerpo del PR + commits.

Registros en vivo: "Ver sesión" → visor de registros en tiempo real.

Al terminar: evento "Copilot finished work" + solicita tu revisión.

Iterar: menciona @copilot en comentarios del PR para solicitar cambios.

A — Aprobar + C — Combinar

PRs siempre en borrador

↩ reglas

Los PRs del agente siempre son borradores hasta que los aprueba un usuario con acceso de escritura.

Los flujos de Actions NO se ejecutan automáticamente → clic en "Aprobar y ejecutar flujos de trabajo".

El solicitante NO puede aprobar su propio PR del agente.

Las reglas de "revisiones requeridas" del repositorio permanecen intactas.

Personalización del entorno y extensión con MCP
copilot-setup-steps.yml

Preinstalar herramientas y dependencias

↩ detalles

Crear en .github/workflows/copilot-setup-steps.yml en la rama predeterminada.

Job obligatorio: copilot-setup-steps.

Solo Ubuntu x64; no ejecutores autohospedados.

Para ejecutores grandes: runs-on: ubuntu-4-core.

Para Git LFS: actions/checkout@v5 con lfs: true.

Firewall: personalizable por directiva organizativa.

MCP — Extensión

Protocolo de contexto de modelo

↩ servidores predeterminados

Estándar abierto para conectar LLMs a herramientas y datos externos.

GitHub MCP: acceso a issues, PRs y datos con token de solo lectura.

Playwright MCP: lee, interactúa y hace capturas de pantalla de páginas web (localhost/127.0.0.1).

Solo admite herramientas (no recursos ni avisos). No se admiten servidores remotos con OAuth.

Validación y uso responsable

Antes de combinar el PR del agente

↩ prácticas

Ejecutar CI (pruebas, linters, análisis) en cada PR del agente.

Inspección manual de áreas sensibles o de alto impacto.

Pedir al agente generar pruebas (consume PRUs).

Aplicar conjuntos de reglas: pruebas + análisis + linting obligatorios antes de merge.

Etiquetar PRs del agente (agent-refactor, agent-tests) para trazabilidad.

Iterar instrucciones en .github/copilot-instructions.md cuando veas errores repetidos.

Solución de problemas frecuentes
Copilot no aparece en Asignados

↩ solución

Verifica que estás en un plan apto (Pro, Pro+, Business, Enterprise).

Confirma que no esté deshabilitado en org o repositorio.

Revisa github.com/settings/copilot/features.

Issue asignado, sin progreso

↩ solución

Actualiza la página; busca la reacción 👀 en el issue.

Verifica la línea de tiempo del PR: "Copilot ha comenzado a trabajar".

Abre "Ver los registros de sesión" para diagnóstico.

Sesiones con tiempo de espera: se agotan tras 1 hora; reasigna el issue.

Actions no se ejecutan

↩ solución

Haz clic en "Aprobar y ejecutar flujos de trabajo" en el cuadro de fusión del PR.

Recuerda: el solicitante no puede aprobar su propio PR del agente.

CI falla / cambios no pasan

↩ solución

Añade instrucciones claras en .github/copilot-instructions.md.

El agente puede autovalidarse con tests/linters si las instrucciones lo especifican.

Advertencias de firewall: ajusta a personalización si necesitas acceso externo.

Resumen

Cloud Agent BFPDD: asistente asíncrono en GitHub para Bugs, Features, Pruebas, Docs y Deuda técnica. Disponible en Pro+, Business, Enterprise.
Seguridad GERPC: Gobernanza org, Entorno sandbox con firewall, Ramas solo copilot/, Permisos de escritura requeridos, Coautoría y aprobación independiente obligatoria.
Riesgos III: Inserción de código, Información confidencial, Inyección de prompts — todos mitigados por diseño.
Ciclo ARIAC: Asignar issue/chat → Rastrear (👀, PR borrador, logs en vivo) → Iterar con @copilot → Aprobar (no el solicitante) → Combinar tras CI.
Personalización: copilot-setup-steps.yml + ejecutores grandes + LFS + MCP (GitHub + Playwright) + copilot-instructions.md.
